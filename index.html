// how to read it

A reading approach that actually works

Most people read programming books passively and retain very little. This is the approach that works — active, daily, code-first.

1

Read the day's section the evening before

Each daily lesson is designed to be read in 30–40 minutes. Read it the evening before you intend to practise — your brain processes new information overnight. You will sit down the next morning already familiar with the concepts.

evening read · 30–40 min
2

Type every code example — do not copy-paste

Typing code by hand, even code you are copying from the book, builds muscle memory and forces you to notice every character. When you make a typo and the program fails, you learn more from fixing it than from reading ten more pages.

type, run, observe · ~45 min
3

Attempt the exercises before looking at the solutions

Give yourself a genuine attempt at each exercise — even if you get it wrong. The struggle is where the learning happens. Only check the solution after you have a working attempt, or after 20 minutes of genuine effort.

exercises · ~30–40 min
4

Build the weekly project without looking at the solution first

The two weekly projects are your checkpoint — they test whether you have truly understood the week's material. Start the project, design your approach, write your code. Only consult the solution when you are stuck, or to compare approaches once finished.

end of week · 2–3 hrs
5

Keep all your code in one folder

Create a folder called python_learning/ on your desktop before Day 1. Save every file you write inside it. By Day 20 you will have a portfolio of working programs you built yourself — proof that you can code.

every day · ongoing
